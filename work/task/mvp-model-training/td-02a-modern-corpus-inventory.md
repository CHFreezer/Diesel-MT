# task TD-02A: 建立近期平行语料来源与授权清单

状态：in_progress

依赖：TD-01；旧 schema v4 与 TD-04 v1/v2/v3 只作为不可变诊断证据

## 目标

为首个约 60M、必须达到翻译及格线的 MVP 重新建立 human-parallel-first 来源清单。优先寻找内容年代可证明为 2020 年以来或由上游持续维护、能够覆盖近年实体与词汇的真实平行语料；不再把“单语 source 交给 Hy-MT2 补齐 20 路”作为默认数据路线。同一 registry 还要判断每个来源的真人 source/target 单语侧能否成为未来独立 64k tokenizer 的候选输入，避免本轮下载、许可审计和清洗成果只能使用一次。

## 原子边界

本 task 只研究来源、版本、时间、许可和可获取范围，不执行全量下载、不调用 DeepSeek、不生成译文、不发布训练 corpus、不训练或冻结 64k tokenizer、不访问 FLORES devtest/正式 test。标记为 tokenizer 候选只表示值得在未来重新评估，不等于已经准入正式 tokenizer 语料。

项目与未来权重按非商业研究用途设计，这会扩大可考虑的 CC BY-NC、CC BY-NC-SA 或明确 research-only 来源，但“项目非商业”不等于任意网页或字幕都可训练。每个来源仍须单独记录训练、衍生权重、数据再分发、署名、相同方式共享和用途限制；结论是项目的数据治理判断，不替代法律意见。

## 首批候选

| 来源 | 主要价值 | 初始角色 | 主要风险 |
| --- | --- | --- | --- |
| OpenSubtitles v2024 | 近年影视、生活口语、网络表达，路线覆盖广 | 按作品/内容年份筛选的对话补充 | 大量重复、错轴、错译、机器字幕、内容年份与发布版本不等价 |
| translatewiki 2025 快照 | UI、平台、账户、现代软件术语，含简繁与非英语直连 | 小比例现代术语层 | 技术域偏置、短片段、占位符与上下文不足 |
| Mozilla 本地化 | 浏览器、隐私、安全、同步、云服务词汇 | 小比例现代术语层 | 上游快照日期与 locale 映射须锁定 |
| MDN Web Docs 2023-09-25 | Web/API/浏览器术语，许可信息明确 | 技术补充 | Hant 数量少、技术比例不能主导 |
| Wikimedia Content Translation | 新实体、人物、产品和百科主题 | 实体/百科补充 | 人机混合翻译、中文简繁需重新分类、部分路线规模小 |
| ALT、KFTT human pair、韩英新闻、UNPC | 已锁定的人类平行锚点 | 历史/正式语域锚点 | 年代较旧；KFTT 只能使用锁定 human pair，不能再让 teacher 猜实体 |
| HPLT/MultiHPLT | 发现近年单语词汇和候选主题 | 词汇发现，不直接计平行语料 | 网页噪声、来源粒度与授权异质、没有天然 target |

候选来源通过官方数据卡、上游仓库或 OPUS 的分 corpus/分版本入口定位；不得把 `OPUS`、`latest` 或聚合包名称当作统一许可和稳定身份。

## 来源记录

每个候选至少记录：

- `source_id`、上游项目、数据卡/下载 URI、具体版本或 commit；
- `snapshot_date` 与 `content_date/content_year` 的可用性和可信度；
- 原始语言/locale、拟映射模型标签、覆盖关系与原始规模；
- 许可证文本/URL、署名和再分发要求、非商业/研究限制、权重发布风险；
- 预计领域、近年词汇价值、重复/错配/机器翻译风险；
- 两侧真人文本是否可独立提取、是否保留文档/作品边界，以及 `tokenizer_candidate_status=allow/hold/reject` 与理由；许可结论必须分别覆盖模型训练、文本处理、衍生产物和数据再分发边界；
- 下载大小、解压大小、文件格式和是否具备稳定 byte lock；
- 决策：`pilot`、`hold` 或 `reject`，以及明确理由。

## 完成条件

- 10 组无向标签关系均至少有一个可进入 pilot 的来源；不足关系明确列为缺口，不以合成数据静默补齐。
- 简体、繁体和粤语边界明确；generic `zh/cmn` 不经证据不能归入任一中文标签。
- 所有 pilot 来源具有稳定版本/时间证据和许可结论；不确定来源保持 `hold`。
- registry 按五标签估算可供未来 tokenizer 复审的唯一真人文本/字符量、年代和领域覆盖；synthetic、机器译文、dev/test 与 tokenizer holdout 不得计入可训练候选量。
- 发布候选 registry 与来源决策报告，供 TD-02B 使用；旧 schema v4 lock 和运行根完全不变。
