# task TD-04: 使用 DeepSeek 长上下文批量审计平行语料

状态：pending

依赖：TD-02B 校准合同、TD-03 preaudit corpus

## 目标

在不为每条句对单独请求或输出 `pass` 的前提下，将大量带稳定 ID 的 source-target 句对合成长上下文批次交给 DeepSeek 扫描，只稀疏返回疑似有问题的 ID、严重度、类型和短理由。以人工样本和已知错误 canary 校准长上下文召回率，降低纯规则清洗的语义漏检和逐条 API 成本。

## 原子边界

DeepSeek 在本 task 仅作辅助审计：不得自动重译、补齐、润色或就地修改原始句对，不得把未返回 ID 解释为逐条模型认证。正式 test、FLORES devtest 和训练 checkpoint 均不得发送到远程 API。

## 批次协议

- 批次优先保持同一路线、相近来源/领域和相近长度；由 token 上限控制，不以固定句数假设上下文容量。
- 输入包含 `batch_id`、任务说明和多条 `{sample_id, source_text, target_text}`；提示词要求检查对齐、遗漏/增译、实体/音译、数字/日期/单位、否定/主体、术语、语言/脚本、繁中地区表达、源文损坏和自然度。
- 输出为紧凑结构化 flag 列表，只含问题 `sample_id`、`severity`、`category`、`reason`；正常记录不回显。响应必须绑定 `batch_id` 和输入计数，禁止只返回自然语言总结。
- prompt 前缀、字段顺序和错误类别表保持稳定，正文使用紧凑 JSONL/TSV，尽量利用服务端可用的前缀缓存；成本报告分开记录输入、缓存命中输入和输出 token。长上下文仍按实际输入 token 计费，不能把减少请求次数误报为零成本。
- 每个批次保存输入 ID/内容哈希、模型/endpoint、prompt、模式、token/费用、原始响应和解析结果；API key 只从 Git-ignored 本地环境读取。
- 网络/限流错误可按冻结重试策略恢复；响应截断、缺少 batch 身份、未知 ID、重复/越界 ID 或 JSON 结构错误使该批次未完成，不能静默当作零问题。

## 校准与全量审计

1. 用 TD-02B 冻结的多个 token 档位跑相同盲校准集，比较 canary 召回、真实坏样本召回、flag precision、未标记人工抽检漏检和单位成本。
2. 选择满足预注册门槛的最大安全批次，而不是盲目占满宣传上下文；如果长上下文导致召回明显下降，采用较小批次。
3. 全量运行只输出 sparse flags；全部 flag 人工复核，未标记记录按来源/路线/年份/领域分层抽检。
4. 高风险 flag 进入 quarantine；误报可由人工恢复。DeepSeek 给出的修正文本即使存在也一律忽略。

## 模型、质量与费用门

- 默认审计模型显式写为 `deepseek-v4-flash`，不使用即将弃用的旧别名；每次正式执行前重新读取官方模型、上下文和价格信息并写入运行 manifest。`pro`/thinking 模式只可用于有界疑难复核，不能静默替换全量审计模型。
- 首轮校准至少比较约 64K、128K、256K input-token 档位；只有更大批次仍满足召回和结构完整性时才可继续上探，不能因服务支持 1M context 就默认填满。
- 全量放行门暂定为：critical 实体/数字/否定/错语言 canary 召回率不低于 95%，flag 人工有效命中率不低于 70%，未标记分层抽查中的严重错误率不高于 1%。任一项失败时缩小批次、修订提示或停止，不能靠追加预算掩盖质量问题。
- pilot/校准预算 10～30元；90万～130万 accepted 规模对应累计 150～350元；扩展到 150万～200万对应累计 300～600元。1000元是悲观硬上限，不是默认可用额度；预计累计超过600元前必须报告原因并获得确认。
- 预算按正文大部分 cache miss 估算。每个批次必须记录 cache-hit input、cache-miss input、output、retry、二次扫描、每万条费用和剩余额度；达到阶段上限时安全停止并保留可恢复 journal。

## 实现约束

优先扩展现有 `deepseek_translation_review.py` 的共享协议或提取公共库，并通过统一 CLI 暴露 batch-audit 子命令；不新增每来源一个脚本。实现必须支持成本上限、原子 journal、断点恢复、批次身份匹配和稀疏结果校验。

## 完成条件

- 校准报告证明所选批次满足 TD-02B 冻结的 critical canary/真实错误召回、precision、未标记漏检和费用门槛。
- 所有 preaudit 样本恰好属于一个完成批次或明确失败/隔离批次；没有响应的批次不能进入 TD-05。
- flag、人工决定、未标记抽检和费用按路线/来源/年代/领域可查。
- 输出只是一份 audit overlay，不改写 TD-03 原始或规范文本。
