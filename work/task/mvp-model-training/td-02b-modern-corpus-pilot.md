# task TD-02B: 运行近期语料小样本实收率与预算试验

状态：pending

依赖：TD-02A

## 目标

对 TD-02A 的 `pilot` 来源执行有界下载和确定性抽样，回答“真正能留下多少、覆盖哪些路线、年代和领域、需要多少磁盘/API/人工成本”。正式规模由实收率决定，不先按漂亮的行数配额反推放宽质量门。

## 原子边界

只运行小样本/小分片 pilot 和人工抽检，不全量抓取、不发布训练 corpus、不进行 DeepSeek 全量审计、不生成 synthetic target、不访问正式 test。

## 执行事项

- 为每个来源锁定一个有界 pilot 分片或确定性样本，记录下载、解压、缓存和预计全量空间。
- 将“数据快照较新”和“文本内容较新”分开统计；有内容年份时报告 2020 年以来占比，没有年份时不得推断。
- 运行与正式 TD-03 相同的廉价硬门：编码、语言/脚本、简繁/粤语、长度、零截断、HTML/模板、source-copy、数字格式、exact/near duplicate 和评测污染。
- 按来源、路线、年份、领域报告 raw、硬门 accepted、唯一 semantic groups、唯一 source/target、tokens 和主要拒绝原因。
- 对硬门 accepted 做有界人工抽检，重点检查对齐、实体、数字、否定、术语、字幕错轴和上下文残缺；此处只估计风险，不替代 TD-04。
- 使用冻结 `mvp-tokenizer-v0` 编码全部 pilot accepted，按标签/来源/领域报告 `<unk>` token、offset 对应原文字符损失、tokens/非空字符、P50/P95/P99/Max、超过候选训练长度的比例及合法实体/姓名字符；不得因总体均值正常而掩盖 Hant 或单一路线退化。
- 对每个来源的真人 source/target 侧分别按稳定文本 ID 去重，报告五标签唯一文本数、清洗后字符数、年代/领域分布、文档边界可用性和未来 64k tokenizer 候选资格；反向路由展开不得重复增加候选量，synthetic、dev/test、canary、quarantine 和 tokenizer holdout 一律排除。
- 根据实测结果冻结最小训练 tranche、可选扩展 tranche、逐来源/领域 ceiling、路线采样策略、下载/解压/缓存预算和 DeepSeek API 费用上限。
- 近期内容层以 20%～25%、持续本地化/技术术语层以 5%～10% 作为 pilot 比较区间，不在看到实收率前把区间强制改成行数 KPI。字幕、技术、法律/政务分别设置 ceiling；Hant 继续质量实收，不为路线对称强行回填。

## 预注册规划区间（由 pilot 实收率确认或修订）

- pilot 先处理约 5万～10万条候选；首轮规划为 90万～130万个独立 human parallel groups/pairs，正反展开约 180万～260万条 directed records；可选扩展上限为 150万～200万个 pairs、300万～400万条 directed records。
- 10 组无向关系的首轮 pairs 区间为：EN–Hans 15万～20万、EN–Hant 6万～10万、EN–JA 15万～20万、EN–KO 15万～20万、Hans–Hant 4万～8万、Hans–JA 8万～13万、Hans–KO 8万～13万、Hant–JA 4万～8万、Hant–KO 4万～8万、JA–KO 8万～13万。
- 主领域初始比较目标为：近期字幕/自然对话 25%、新闻/百科/一般说明文 25%、高可信人工锚点/正式文本 20%、日常社区短句 10%、持续维护 UI/本地化 10%、现代技术文档 5%、简繁互转/地区正式繁体 5%。单一来源原则上不超过 30%，技术域全局约 5%，Hant 技术域不超过 15%，Hant 法律/政务不超过 20%。
- 新鲜度单独统计：可证明 2020 年以来的内容目标 20%～25%，持续维护但单句年份未知的本地化目标 5%～10%，其余允许来自较旧或年代未知但质量可靠的来源。
- 上述全部是容量与风险预案，不是必须填满的 quota。若高质量实收不足，输出 `revise/stop` 并回到 TD-02A，不得通过放宽许可、语义或 Hant 质量门凑数。

## DeepSeek 校准输入

从 pilot accepted 中建立 TD-04 使用的盲校准集：

- 覆盖五标签、主要来源、年代和领域；
- 包含人工确认的真实好/坏样本；
- 另生成不进入训练的可追踪 canary，只注入实体、数字、日期、否定、遗漏、增译、目标语言和现代术语等已知错误；
- canary 与真实样本分账，任何阶段都不得发布到训练 corpus。

费用预案按 `deepseek-v4-flash` 当前 cache-miss 主体价格估算：pilot/上下文档位校准 10～30元；首轮 corpus 的累计审计预算 150～350元；扩展 tranche 的累计预算 300～600元；1000元为悲观硬上限而非默认授权，预计超过600元时必须先重新报告并确认。执行前重新核对官方价格，并按实际 input cache hit/miss、output、retry 和二次扫描分账。

## 完成条件

- 每个 `pilot` 来源都有实测可用率、唯一语义规模、日期/领域分布、空间和许可复核结果。
- `mvp-tokenizer-v0` 对新语料有唯一 `pass/block` 结论；`pass` 保持 49,152 内容和 ID 不变，`block` 只能创建后续版本化 tokenizer 决策，不在本 task 修改产物。计划中的 64k tokenizer 属于 MVP 通过后的约 201.5M 正式基线，不能用来绕过本门槛。
- 输出独立的未来 tokenizer 候选收益表，能回答本轮每下载/清洗一个来源可沉淀多少唯一真人单语字符及其质量结构；该表不是 64k 配置、训练集或完成身份。
- 冻结 TD-03 全量扫描上限与存储预算，以及 TD-04 的批量大小候选、人工抽查规模、canary 召回和未标记漏检门槛。
- 暂定全量审计解锁线为 critical 实体/数字/否定/错语言 canary 召回率不低于 95%、flag 人工有效命中率不低于 70%、未标记分层抽检中的严重错误率不高于 1%；pilot 可提高门槛，但不得降低后直接放行。
- 给出单一 `proceed/revise/stop` 决策；若预计可用规模不足以支撑 60M MVP，回到 TD-02A 扩展来源，不启动全量构建。
