# task TD-16D: 验证重复训练能力等价并冻结唯一候选

状态：pending

依赖：TD-16C

## 目标

验证最终训练配方在重复训练时产生统计上能力等价的模型，并依据冻结 human dev 规则选出唯一候选；不要求模型 hash 一致。

## 原子边界

本 task 不修改数据、配方或预算，不访问 test，不为了让失败复跑通过而追加训练。只执行 TD-16A 冻结的重复次数和能力等价判定。

## 执行事项

- 对选定配方执行独立重复训练，记录 seed、环境、墙钟、累计 token 和最佳 checkpoint。
- 比较总体及20路 chrF/SacreBLEU、dev loss、脚本合规、空输出、source copy、目标控制和达到质量门槛的时间分布。
- 允许权重、逐步 loss 和 checkpoint hash 不同；输入/配置/checkpoint 文件完整性仍必须可验证。
- 若重复运行超出任一预先冻结能力容差，配方退回 TD-16A 重新设计，不能选择表现最好的一次掩盖不稳定性。
- 通过后只按 dev 冻结唯一候选和不可变 test 授权记录。

## 产物与验收

- 重复训练能力等价报告、time-to-quality 分布、唯一候选和正式 test 授权。
- 唯一候选在总体与20路指标上满足冻结容差，正式 test 消耗仍为 `0/1`。
