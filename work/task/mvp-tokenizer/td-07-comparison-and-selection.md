# task TD-07: 32k vs 48k 对比与 MVP 默认选定

状态：pending

依赖：TD-05（覆盖率报告）、TD-06（保存与加载验证）

## 目标

汇总两份候选 tokenizer 的全部量化指标，选定 MVP 默认候选并记录选择理由。

## 输入

- [mvp tokenizer todo](../../todo/mvp-tokenizer.md)
- TD-05 覆盖率报告和对比摘要
- TD-06 保存/重载一致性验证记录
- README MVP 模型配置表（`e8-d2-v32k`、`e8-d2-v48k`、`e12-d3-v32k`、`e12-d3-v48k`）

## 执行事项

- 汇总两份候选的全部指标：`<unk>` token 比例、原文字符丢失率、频率加权覆盖率、唯一字符覆盖率、字符到 token 膨胀比、平均 token 数。
- 按语言分别比较，标注差异较大（>5%）的指标。
- 按文字系统比较：CJK 汉字切分一致性、韩文音节覆盖完整性、英文 subword 粒度合理性。
- 考虑词表大小对下游模型参数量的影响（32k vs 48k 在四个 MVP 配置下的 embedding 参数差异约 8M–12M）。
- 选定 MVP 默认候选并记录选择理由和实施权衡。理由必须引用 TD-05 和 TD-06 的量化数据，不能仅凭主观偏好。
- 若 32k 和 48k 各有优势场景，明确记录各自推荐使用条件（例如"48k 更适合 CJK 为主的混合语料，32k 在 e8-d2 极小模型上有参数效率优势"）。
- 记录非默认候选的保留位置和用途。

## 产物

- 候选对比报告（`artifacts/tokenizers/reports/selection.md`）
- MVP 默认选定记录（含量化依据）

## 验收

- 选定有明确的量化依据，引用了 TD-05 和 TD-06 的数据。
- 非默认候选的保留位置和备选用途已记录。
- 若 32k 和 48k 各有优势场景，推荐条件已明确。

## 验证记录

### 50% 四语预选（2026-07-14）

- 同一 50% checkpoint provenance 下，48k 相比 32k 将总体 tokens/char 降低 `14.89%`，四个语种均缩短 `12.33%`–`16.20%`。
- 原文字符丢失率均为 `0.001275%`，频率加权覆盖率均为 `99.998725%`，唯一字符覆盖率均为 `99.879032%`，精确 roundtrip 均为 `99.65%`。
- 48k 的序列收益超过预设 10% 阈值，因此选为加入原生繁体和收紧清洗后的一次性重训规模；32k 保留为工程回退和下游吞吐对照。
- 本记录不是最终 MVP 冻结：当前候选没有原生 `zho_Hant`，TD-05/TD-06 仍需在五语候选上完成。最终 `selection.md` 和任务状态保持待完成。
