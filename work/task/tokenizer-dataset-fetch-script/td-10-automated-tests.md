# task TD-10: 建立自动化测试

状态：pending

启动依赖：TD-04

完成依赖：TD-05 至 TD-09

## 目标

以无网络 fixture 和受控集成测试覆盖下载链路、稳定复现契约及主要失败模式。

## 输入

- TD-03 至 TD-09 的模块和 schema。
- 四语 fixture、模拟 map 和模拟分片。

## 执行事项

- 覆盖 registry、lock、map 解析、语言映射和配置校验。
- 覆盖文本清洗、reason code、精确去重、MinHash 和字符预算。
- 覆盖缓存命中、缓存损坏、网络失败、中断恢复和原子输出。
- 在两个全新目录构建并逐文件比较字节及 SHA-256。
- 分别使用冷缓存、热缓存和不同并发度构建，比较输出哈希。
- 验证远端、缓存或 source lock 校验不一致时明确失败。
- 验证 `--dry-run`、`--offline` 和无效参数行为。

## 并行边界

TD-04 完成后即可建立 fixture、测试框架和 CLI 测试。TD-05 至 TD-09 每完成一个模块，应立即补齐对应测试；TD-10 可以持续处于 `in_progress`，但必须在 TD-09 的报告测试完成后才能进入 review。

## 产物

- 自动化单元测试与集成测试。
- 可在本地快速执行的 fixture。
- 标准测试命令和结果记录。

## 验收

- 测试不依赖完整 HPLT 或公网即可覆盖核心逻辑。
- 稳定复现、错误恢复和校验失败测试全部通过。
- 测试运行不会向 Git 跟踪路径写入大文件。
- 失败信息能定位到来源、分片或处理阶段。
