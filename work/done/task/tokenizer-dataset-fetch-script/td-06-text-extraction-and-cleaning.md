# task TD-06: 实现文本抽取与保守清洗

状态：done

依赖：TD-04

## 目标

从 HPLT 文档稳定抽取可训练文本，执行不改变语言文字形态的保守清洗，并产生可审计统计。

## 输入

- HPLT JSONL 文档 schema 和四语 fixture。
- 配置中的长度与清洗规则。

## 执行事项

- 抽取正文、来源 URL、来源 ID、语言和必要质量元数据。
- 明确文本单元切分规则，并固定空白与换行处理。
- 过滤空文本、明显乱码、HTML 残留和超出长度边界的文本。
- 保留大小写、简繁、假名和韩文原始形态。
- 不执行依赖系统 locale 的隐式编码或 Unicode 转换。
- 为每条拒绝路径定义稳定 reason code，并累计输入、保留和拒绝数量。

## 产物

- 文本抽取与清洗模块。
- 版本化清洗规则及 reason code。
- 四语清洗测试和过滤统计。

## 验收

- 相同输入始终产生相同文本和 reason code。
- 输出文本为有效 Unicode，规则不会改变目标文字形态。
- 四种语言的保留与拒绝样例均有测试覆盖。
- 清洗统计之和与输入数量一致。

## 验证记录（2026-07-13）

- `conservative-v3` 抽取正文与来源 URL，按源行切分，统一水平空白；不做小写化、简繁转换、假名/宽度转换或韩文罗马化。
- reason code 覆盖空文本、过短/过长、替换字符、控制字符、HTML、模板及关键词堆砌；高置信模板规则要求商业动作共现或至少五个同类 SEO/成人标记，避免按主题泛化删除。
- 四语 fixture 验证原文字形保持、每个拒绝路径和自然语境保留；最终 report 中每种语言的输入、清洗、过滤统计完整可审计。
