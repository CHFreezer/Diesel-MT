# 50% 四语 tokenizer 预选结论

日期：2026-07-14

## 结论

在使用相同语料、checkpoint provenance、采样比例和评测集的前提下，48k 在字符覆盖和原文丢失完全不变的情况下显著缩短四语序列，因此选为下一轮五语一次性重训的默认词表规模。

该结论不表示当前四语 48k 已经冻结为最终 MVP tokenizer。项目仍需加入原生 `zho_Hant`、收紧清洗、建立繁体固定评测集并重新训练 48k。

## 量化依据

| 指标 | 32k | 48k | 48k 相对变化 |
| --- | ---: | ---: | ---: |
| Tokens/非空字符 | 0.5820 | 0.4953 | -14.89% |
| 平均 tokens/样本 | 205.37 | 174.78 | -14.89% |
| P95 token 长度 | 610 | 519 | -14.92% |
| P99 token 长度 | 1550 | 1303 | -15.94% |
| 原文字符丢失率 | 0.001275% | 0.001275% | 相同 |
| 字符频率覆盖率 | 99.998725% | 99.998725% | 相同 |
| 唯一字符覆盖率 | 99.879032% | 99.879032% | 相同 |
| 精确 roundtrip | 99.65% | 99.65% | 相同 |

各语种 tokens/字符降幅：英语 `16.20%`、简体中文 `12.33%`、日语 `15.61%`、韩语 `15.80%`。

48k 比 32k 在 `d_model=512` 时增加约 8.39M 个共享 embedding 参数，并扩大输出 softmax；该成本需要在五语 tokenizer 冻结后通过真实 MVP 模型 tokens/s、峰值显存和截断率继续验证。32k 产物保留为工程回退及该吞吐对照。
